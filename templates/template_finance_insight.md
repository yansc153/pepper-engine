# 金融干货模式 / Template Finance Insight

字数：≤ 280
目标：把财报数据、宏观信号、研报观点改成有深度判断的中文金融推文。

---

## 定位

「干货」模式 = 数据驱动 + 框架驱动 + 预期差驱动
不是段子，不是情绪，而是「这个数字背后的真实含义」。

适用场景：
- 财报发布后的关键数据解读
- 宏观数据（CPI、M2、成交额、库存）的交易含义
- 板块行情的基本面支撑或破坏信号

---

## 结构骨架

```
[第一句：这个数字最值钱的地方]

[证据链 1：数字 / 比值 / 同比 / 环比]

[证据链 2：这个数字被市场忽视的地方]

[我的结论：对谁有利，对谁不利]

[观察位：后续关键数字或时间节点]
```

---

## 推荐结构

### 结构 1：财报数字反转预期

```
财报的 headline 是 X

但这个细节改变了我的看法

[数字 + 含义]

[对标去年同期，是向好还是向坏]

[这意味着接下来要关注什么]
```

**例子**：
```
利润增长 30% 看起来不错

但毛利率掉了 3 个点

说明价格战还在继续

下个季度的指引能不能接住这个估值是关键
```

### 结构 2：数据背后的分化

```
整个数据看起来 [向好/向坏]

但分拆来看，[某类票/某个板块] 跟不上

原因是 [具体的数字证据]

[我的立场]

[盯这个关键指标确认趋势]
```

**例子**：
```
全 A 成交额 3 万亿

但创业板成交额环比下滑

说明增量主要是蓝筹

如果明天创业板还不跟，就要当心分化加剧

先盯明天的创业板成交占比
```

### 结构 3：预期差的落差

```
市场以为 [主流共识]

但数据显示 [真实情况]

[具体数字对比]

这对 [谁] 有利

[观察位]
```

**例子**：
```
市场以为高利率是利空

但银行股还在涨

原因是真实利差扩宽了

这对银行的净息差有利

看下个季度的存款成本能不能继续走低
```

### 结构 4：时序对比揭示真相

```
[时间 A] 是这样的

[时间 B] 是那样的

比差值 / 增速 / 涨幅

[对交易的启示]
```

**例子**：
```
2024 年初，白酒库存在 500 万吨

现在库存掉到 380 万吨

这意味着 Q2 可能没有那么惨

不过也要看成交额有没有真的起来
```

## 关键要素 Checklist

- [ ] 有具体数字（财报数据、百分比、环比、同比、绝对值）
- [ ] 数字来自可核实的来源（财报、公告、交易所数据）
- [ ] 有「市场可能忽视」的那一层
- [ ] 有对标对象（去年同期、行业平均、竞争对手）
- [ ] 有明确的交易含义（对谁有利、对谁有害、板块排序变化）
- [ ] 有后续观察位（什么时间、什么数据、什么指标）
- [ ] 没有预测性词语（「应该」「可能」「大概」）
- [ ] 没有无来源定性（「市场普遍」「业内认为」）
- [ ] 没有喊单迹象（「建议」「推荐」「我看好」）

---

## 反面教材

**不好的干货文**：
```
这家公司的财报很不错

利润增长比较快

估值有修复空间

值得关注
```

问题：全是空话，没有数字，没有观察位

**好的干货文**：
```
利润增长 35%，但毛利率掉了 2 个点

说明量增但价跌

今年的指引没变，明年的指引下调了 10%

这票要看 Q3 能不能在成本上扭转

先盯下月的库存数据
```

---

## 写作节奏

1. 找最硬的一个数字
2. 问「这个数字被市场怎么理解的」
3. 问「这个数字的另一面是什么」
4. 写出预期差
5. 加上交易含义
6. 收尾观察位

---

## 禁止项

不用这些词和句式：
- 「值得关注」「具有意义」「重点考虑」
- 「据了解」「有报道」「相关人士」
- 「可能」「应该」「大概」「或许」
- 「建议」「推荐」「我看好」「不错」
- 「未来前景」「长期看好」「估值修复空间」

用具体数字和观察位替代。
