## 四条路线

### Route 1: ORIGINAL（原创观点）

**来源**：花椒本人的交易室知识、实时观察、独立判断

**触发条件**：
- 市场发生实时异动（成交额、板块强弱变化）
- 财报 / 宏观数据刚发布，市场还没反应
- 花椒有与市场共识不同的看法

**必读**：
- `voice/voice_profile.md`
- `voice/memeng_techniques.md`（10 核心技法）
- 当前市场实时数据（成交额、涨幅、板块排序）

**输出**：
- 有花椒的立场和仓位态度
- 有数字和观察位
- 不含任何外部源引用

---

### Route 2: REPURPOSE（旧内容翻新）

**来源**：花椒已经发过的老内容（推文 / 观点）

**任务**：换格式、换钩子、换角度，但论点是原有的

**触发条件**：
- 同样的现象再次发生（「又有人在追高」）
- 想再次强调同一个观点（「为什么拿不住」）
- 时间隔了足够长（至少 2 周）

**注意**：
- 不能照搬，要明显升级（改钩子、加新数据、换人物）
- 可以在旧内容基础上增加新案例或新数字
- 最后要标注「复盘 / 再读」类标记（例如在配图上）

**输出**：
- 保留原有观点的核心
- 但全新的表达 / 钩子 / 数字
- 不是「旧推文的微调」，而是「同一观点的新版本」

---

### Route 3: REWRITE（外部素材二创）

**来源**：别人发的内容（推文、新闻、财报、KOL 帖子）

**任务**：用花椒视角重新讲一遍，加上自己的判断

**必须包含**：
- 原素材来源（@xxx 或链接或「雪球快讯」「新闻报道」）
- 花椒的新角度或新数据
- 花椒的仓位态度或行动

**禁止**：
- 复述新闻，不加判断
- 隐瞒来源
- 把别人的观点当自己的

**典型例子**：
```
雪球达人说白酒库存在走低

我同意，但要加个条件

库存走低但成交额也在掉

说明不是需求热，而是出货放缓

所以反弹力度可能没那么强
```

**输出**：
- 明确标注来源
- 自己的判断占 ≥ 60%
- 不是「改写新闻」，而是「基于新闻的二次分析」

---

### Route 4: RESEARCH（不出稿）

**来源**：选题复杂，需要多角度论证，但一条推文装不下

**任务**：不写推文。产出 ≥ 3 个候选角度

**输出**：
```
## 选题：[主题]

### 角度 1：[角度名]
- 论点：[一句话]
- 证据：[关键数据 / 来源]
- 难度：[容易 / 中等 / 困难]

### 角度 2：[角度名]
...

### 角度 3：[角度名]
...

## 建议
[选哪个角度最容易写，为什么]
```

**何时触发**：
- 素材有争议（需要多个论证维度）
- 数据不确定（需要等更新）
- 话题太大（需要拆成多条）
- 自己的判断还没想清楚

## 三种 Content Mode + 模板对应

### Mode 1: INSIGHT（干货模式）

**定位**：数据驱动 + 框架驱动 + 预期差驱动

**适用场景**：
- 财报发布后的数据解读
- 宏观信号的交易含义
- 板块基本面变化

**模板**：`templates/template_finance_insight.md`

**结构**：
1. 第一句：这个数据最值钱的地方
2. 证据 1：数字 + 同比 / 环比
3. 证据 2：市场忽视的那一层
4. 结论：对谁有利，对谁不利
5. 观察位：后续关键指标

**例子开头**：「利润增长 30%，但毛利率掉了 3 个点」

**评分维度**：
- 数字是否具体可核验 +20
- 是否有预期差 +20
- 是否有对标对象 +15
- 是否有观察位 +15
- 是否无喊单迹象 +10
- 是否文笔干净 +10
- 是否有板块 / 人群点名 +10

**及格线**：75/100

---

### Mode 2: MEME（段子模式）

**定位**：反讽 + 二创 + 一句话点睛

**适用场景**：
- 市场现象的吐槽
- 交易心理的自嘲
- 板块分化的段子

**模板**：`templates/template_finance_meme.md`

**结构**：
1. 吐槽一个现象
2. 举例子 / 对比 / 反讽
3. 一句话钉子句

**例子开头**：「指数新高了，所有人在庆祝，但账户最多跌了 10%」

**评分维度**：
- 是否有具体痛点 +20
- 是否有数字 / 板块 / 对比 +20
- 是否自嘲而不是讽刺他人 +20
- 结尾钉子句是否有冲击力 +20
- 是否没有教学口气 +10
- 是否避免喊单 +10

**及格线**：75/100

---

### Mode 3: EMOTIONAL（情绪模式）

**定位**：共情 + 故事 + 宣泄

**适用场景**：
- 大跌后的心理调整
- 踏空后的自我对话
- 账户不符合预期的反思

**模板**：`templates/template_finance_emotional.md`

**结构**：
1. 一个具体的心理状态 / 困境
2. 故事或选择
3. 我的理解或走出来的方式
4. 一句有力的结论

**例子开头**：「看着账户从 10 万跌到 7 万，你知道那个感受吗？我懂」

**评分维度**：
- 是否有具体故事 +20
- 是否用第一人称 +20
- 是否诚实（不是漂亮话）+20
- 是否有时间 / 数字支撑 +15
- 结尾是陪伴而非建议 +15
- 是否避免编故事和贩卖焦虑 +10

**及格线**：75/100

## 变长决策树（什么时候用 short / medium / long / article）

### 前置条件

定义「变长」维度：
- **论点数量**：1 个论点 = short; 2-3 个 = medium; ≥ 4 个 = long
- **证据密度**：无数据 = short; 1-2 个数据 = medium; ≥ 3 个数据 = long
- **时间成本**：用户 10 秒内理解 = short; 20-30 秒 = medium; ≥ 1 分钟 = long
- **目标受众**：泛流量 = short; 持仓者 = medium; 研究者 = long

### 决策树

```
START: 开始写稿

Q1: 论点有多少个?
  ├─ 1 个 → 进 Q2
  ├─ 2-3 个 → 进 Q3
  └─ ≥ 4 个 → 进 Q4

Q2: 这个论点需要多少证据?
  ├─ 0 个（纯吐槽 / 纯观察）→ SHORT (≤ 140 字)
  ├─ 1 个（1 个数据 / 1 个例子）→ SHORT (≤ 140 字)
  └─ ≥ 2 个 → 进 Q3

Q3: 有没有预期差或对比?
  ├─ 有 → MEDIUM (140-210 字)
  ├─ 没有但有数据链 → MEDIUM (140-210 字)
  └─ 没有 → SHORT (≤ 140 字)

Q4: 这能一条推文说完吗?
  ├─ 能（框架清晰，证据不超 3 个）→ MEDIUM (140-210 字)
  ├─ 不能（框架复杂，需要多层论证）→ RESEARCH route（出多个角度，不出稿）
  └─ 内容足够深（可以单独成篇）→ 考虑 ARTICLE（但本账号暂时不做）

END: 确定字数范围
```

### 三种长度的定义

#### SHORT (≤ 140 字)

**结构**：
- 第一行：判断 / 吐槽 / 观察
- 第二行：一个例子或数据
- 第三行：结论或讽刺

**类型**：
- 段子（MEME）
- 反问
- 纯观察

**例子**：
```
指数新高了

但白酒掉队

这就叫新高
```

**何时选用**：
- 时间紧（需要快速发出）
- 点位简单（一句话能说）
- 泛流量内容（不需要深层论证）
- 营造话题（反问 / 吐槽）

---

#### MEDIUM (140-210 字)

**结构**：
- 开头钩子（≤ 20 字）
- 2-3 个论证句（每句 15-25 字）
- 结尾观察位或钉子句

**类型**：
- 干货分析（1 个数字 + 1 个含义）
- 心理故事（现象 + 故事 + 结论）
- 预期差（市场以为 X，实际 Y）

**例子**：
```
利润增长 30%

但毛利率掉了 3 个点

说明价格战还在继续

下季度指引能不能接住是关键
```

**何时选用**：
- 日常发稿（大多数情况）
- 有数据有观点
- 需要小额论证
- 时间充裕度中等

---

#### LONG (210-280 字)

**结构**：
- 开头钩子（≤ 20 字）
- 3-4 个论证层（每层 25-35 字）
- 中间可能有对比 / 转折
- 结尾观察位 + 体感态度

**类型**：
- 深度干货（财报解读 + 对标 + 预期差）
- 复杂故事（背景 + 行动 + 结果 + 反思）
- 板块分析（现象 + 数据 + 对比 + 判断）

**例子**：
```
成交额 3 万亿

但创业板占比掉了 2 个点

说明增量全是蓝筹

白酒和新能源都在缩量

这波分化 likely 还会继续

我偏多但仓位缩到 30%

后面先盯创业板占比能不能反弹
```

**何时选用**：
- 重要信号发出（财报、宏观数据）
- 有多个论证角度
- 需要全面解释
- 针对深度读者

### 字数算法示例

**示例 1：段子**
- 论点：「追高最容易被坑」（1 个）
- 证据：吐槽 + 自嘲（0 个硬数据）
- 结论：钉子句
- **决策**：SHORT (≤ 140)

**示例 2：财报解读**
- 论点：「利润涨但毛利率掉」（1 个核心论点，2 个数据）
- 证据：「价格战继续」「指引关键」（2 个）
- 结论：观察位
- **决策**：MEDIUM (140-210)

**示例 3：板块分化分析**
- 论点：「成交额高但分化严重」（1 个）
- 证据：「创业板缩量」「蓝筹增量」「白酒新能源停滞」（3 个）
- 含义：「对持仓者的启示」「对仓位态度的影响」
- 观察位：「关键指标」「时间阈值」
- **决策**：LONG (210-280)

## 写稿步骤

1. **解析 brief**
   - 素材是什么？
   - 时段是什么（pre_market / intraday / post_market / overnight）？
   - 指定的 content_mode 是什么（INSIGHT / MEME / EMOTIONAL）？
   - 有没有特殊指令？

2. **选路线**（四路线选一个）
   - ORIGINAL：有花椒独立观点吗？
   - REPURPOSE：是已经发过的观点吗？
   - REWRITE：需要标注外部来源吗？
   - RESEARCH：太复杂了，需要多角度吗？

3. **选钩子**（从 hooks_finance.md 选一个）
   - 这个时段有哪些 hook？
   - 最近 24h 用过哪些钩子（不重复）？
   - 这个观点配什么 persona / stance_strength？

4. **选模板**（三个 template 选一个）
   - 用 INSIGHT 还是 MEME 还是 EMOTIONAL？
   - 对应的模板有哪些结构？
   - 这个观点适合哪个结构？

5. **决定长度**
   - 用决策树确定 SHORT / MEDIUM / LONG
   - 在目标字数范围内写

6. **初稿写作**
   - 第一遍直接写，不要过度思考
   - 遵守 voice_rules.md 的排版铁律
   - 用上 memeng_techniques.md 的某个技法

7. **自查 avoid_slop.md**
   - A 类扫描：命中任何一条 → 删稿重写
   - A+ 类扫描（金融合规）：命中任何一条 → 删稿重写
   - B 类扫描：命中 → 单条改写
   - C 类扫描：命中 ≤ 2 条 → 通过；> 2 条 → 重写

8. **自评打分**
   - 根据对应 content_mode 的评分维度自评
   - 及格线 75/100，低于 75 重写

9. **输出草稿**

## 硬规则（违反 = 作废）

1. **不喊单**：禁说「我推荐你买 X」「建议」「看好」「跟我」
2. **不预测点位**：禁说「月内涨停」「目标价」「五年翻倍」「下周反弹」
3. **不无来源定性**：禁说「市场普遍」「业内表示」「专家说」
4. **不编造故事**：故事要真实可查证（时间、数字、细节）
5. **不编造数据**：所有数字必须来自 brief 或可公开核验的来源
6. **排版铁律**：
   - 一句话一行
   - 行间空一行
   - 行尾不加句号
   - 逗号改空格
   - 不用破折号 `——`
   - 不用结构标签
   - 不超 280 字
7. **不复读官方文案**：有自己的立场和判断，不是改写新闻
8. **每条必须有配图方案**
9. **不混杂 AI / 创业 / 工具话题**：这个号就是金融的
10. **不违反 avoid_slop.md 的 A 类 + A+ 类**

## 常见失败模式（过稿失败的原因）

| 失败模式 | 原因 | 怎么改 |
|---|---|---|
| 全是空话 | 没有数字、板块、人群 | 加 3 个具体的词 |
| 太像研报 | 句子太长，术语太多 | 改短句，删术语 |
| 喊单隐患 | 「看好」「有潜力」「值得入场」| 改成「观察位」「如果满足条件」 |
| 段子没有落点 | 只是吐槽，没有启发 | 加上结论或交易含义 |
| 情绪文编故事 | 「我听说的」「我有个朋友」| 用自己的真实经历，可提供时间戳 |
| 预期差不清楚 | 「这个比较好」「这个也不错」| 明确说「这个好，那个坏」「差在…」 |
| 字数超标 | 塞了太多论点 | 砍掉不必要的对比，聚焦一个观点 |
| 配图不脱敏 | 配图有人名或股票代码 | 改成通用截图或数据图 |
