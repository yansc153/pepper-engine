# 内容过滤评分规则 — Finance Main Account

> 内容生成后用这个 rubric 评分，决定发还是不发。

---

## 评分维度（4 维度，总分 0-85）

### Dim 1 — 立场可承接度（0-20）

读 `config/persona.md` §4 核心立场。这条内容是否有"花椒立场可以套上去"的角度？

| 分数 | 标准 |
|---|---|
| 0  | 内容与花椒所有立场都无法挂钩 |
| 5  | 勉强能套，但角度生硬 |
| 12 | 有 1-2 个明显立场切入点 |
| 20 | 同时切多个市场立场（如预期差 + 仓位纪律 + 板块分化） |

### Dim 2 — receipts 可验证度（0-20）

是否有具体数字 / 可验证 source / 可截图的内容？

| 分数 | 标准 |
|---|---|
| 0  | 全是定性描述，无数字 |
| 8  | 有数字但无原始 source |
| 15 | 有数字 + 链向官方/产品页 |
| 20 | 多个 receipts，截图素材丰富 |

### Dim 3 — 反共识/故事性（0-25）

| 分数 | 标准 |
|---|---|
| 0  | 完全合规共识叙事，无张力 |
| 10 | 有一个共识可以反 |
| 18 | 反共识 + 有具体案例 |
| 25 | 反共识 + 故事 + 反转可拆 |

### Dim 4 — 受众相关度（0-20）

| 分数 | 标准 |
|---|---|
| 0  | 受众完全不关心 |
| 8  | 一部分受众相关 |
| 15 | 主受众痛点 |
| 20 | 主受众痛点 + 强情绪共鸣 |

---

## 决策规则

```
total = Dim1 + Dim2 + Dim3 + Dim4  (max 85)

  total < 40          → drop（不发）
  40 ≤ total < 55     → needs_review（改写后重评）
  total ≥ 55          → pass（可以发）
```

- 含有 `voice/avoid_slop.md` A 类词汇 → 直接 drop
